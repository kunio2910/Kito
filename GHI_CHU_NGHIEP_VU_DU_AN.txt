# **GHI CHU NGHIEP VU VA CHUC NANG DU AN BAI GIANG TREN NUI**
------------------------------------------------------------
Cap nhat: 13/07/2026

------------------------------------------------------------
## **1. MUC DICH DU AN**

Website "Bai Giang Tren Nui" la website chia se noi dung Cong Giao, phuc vu viec hoc hoi, cau nguyen va chia se Tin Mung.

Website co 2 nhom nguoi dung chinh:
- Nguoi xem website: doc bai viet, xem cac thanh, nha tho, su kien, giao ly, loi cau nguyen, gui loi cau nguyen, danh gia bai viet va gui y kien dong gop.
- Quan tri vien: dang nhap trang quan ly de them, sua, xoa, an/hien noi dung, duyet loi cau nguyen, xem danh gia, xem y kien dong gop va thong ke truy cap.

## **1A. QUY CHUAN GIAO DIEN HIEN TAI CAN GIU NGUYEN**

Phan nay ghi lai font chu, size chu va khoang cach hien tai trong `styles.css`. Khi chinh sua chuc nang moi, uu tien giu nguyen cac thong so nay de tranh loi giao dien, tru khi nguoi dung yeu cau doi truc tiep.

### **1A.1. Font chu**

**Font toan website:**
- Bien CSS: `--font-main`.
- Gia tri: `Tahoma, "Segoe UI", Arial, sans-serif`.
- Dung cho `body` va phan lon noi dung.

**Font tieu de:**
- Bien CSS: `--font-heading`.
- Gia tri: `"Segoe UI", Tahoma, Arial, sans-serif`.
- Dung cho hero title, section heading, tieu de trang, tieu de card quan trong.

**Font cau trich/cau nguyen:**
- Dung trong `.daily-card blockquote`, `.daily-card small`, `.prayer-card blockquote`.
- Gia tri: `Cambria, "Palatino Linotype", "Times New Roman", serif`.
- Thuong di kem `font-style: italic` cho cau trich.

### **1A.2. Size chu chinh**

**Nen tang:**
- `body`: `14px`.
- Link/nut nho: `11px - 12px`.
- Mo ta card: `12px`.

**Header:**
- Logo/brand text: `10px`, in dam, uppercase.
- Icon logo: `28px`.
- Menu header `.main-nav`: `15px`, font-weight `700`.
- Khung tim kiem header: input `12px`, icon `24px`.

**Trang chu:**
- Hero title: `clamp(36px, 4.2vw, 58px)`.
- Hero paragraph: `13px`.
- Tieu de section `.section-heading h2`: `18px`, uppercase, font-weight `800`.
- Link `Xem tat ca` va link card: `12px`, font-weight `700`.
- Quick link title: `13px`.
- Quick link icon: `26px`.

**Card noi dung:**
- Tieu de card: `14px`, line-height `1.35`.
- Mo ta card: `12px`, line-height `1.5`.
- Event date number: `26px`.
- Event text nho: `11px - 12px`.

**Loi Chua moi ngay:**
- Quote: `13.2px`, line-height `1.6`, italic.
- Ref: `12px`.

**Cau Nguyen:**
- Tieu de card cau nguyen: `15px`, line-height `1.3`.
- Label phu: `11px`, uppercase.
- Trich doan: `12px`, line-height `1.45`, italic.

### **1A.3. Khoang cach va le header**

**Header chung `.site-header`:**
- `min-height: 62px`.
- `padding: 0 18px 0 30px`.
- `gap: 14px`.
- Nen trang chu desktop: header nam trong cot trai, rong `52.4%`.

**Logo/brand:**
- `min-width: 118px`.
- `gap: 8px`.

**Menu header:**
- `gap: 12px`.
- Vach ngan giua cac muc: rong `1px`, cao `14px`, `margin-right: 12px`.

**Header actions:**
- `gap: 10px`.
- `margin-left: auto`.

**Khung tim kiem header:**
- `width: 230px`.
- `min-height: 34px`.
- `padding: 0 12px`.
- `gap: 6px`.
- `border-radius: 999px`.

### **1A.4. Khoang cach va le trang chu**

**Bo cuc trang chu:**
- `.home-dashboard`: 2 cot desktop.
- Cot trai: `52.4%`.
- Cot phai: `47.6%`.
- `.right-column`: `gap: 22px`, `padding: 24px 30px 28px`.
- `.top-panels`: `gap: 22px`, 2 cot, dang stretch de `Loi Chua moi ngay` bang chieu cao `Gioi thieu nha tho`.

**Banner hero:**
- `min-height: 460px`.
- Noi dung hero: `width: min(760px, calc(100% - 32px))`, `padding-top: 22px`.
- Hero title margin duoi: `14px`.

**Quick links:**

- Icon tron: `50px x 50px`.

**Section heading:**
- `.section-heading`: `gap: 16px`, `margin: 0 0 16px`.
- Trong cot phai: `padding: 5px 5px 0`.

**Grid va card trang chu:**

### **1A.5. Kich thuoc card va anh**

**Card chung:**
- Border: `1px solid var(--line)`.
- Border radius: `8px`.
- Shadow: `var(--shadow)`.
- Noi dung card: `padding: 10px`.

**Anh card:**

- Nha tho: `height: 104px`, `object-fit: contain`.

**Daily card:**

**Event card:**

**Prayer card:**

**Shell chung:**
- `.detail-shell`, `.category-shell`: `width: min(100% - 40px, 1320px)`, `max-width: 1320px`.
- `margin: 0 auto`.
- `padding: 28px 20px 46px`.

**Trang chi tiet:**
- `.detail-article`: grid 2 cot `minmax(240px, 360px)` va `minmax(0, 1fr)`.
- `column-gap: clamp(24px, 4vw, 54px)`.
- Border radius: `8px`.

### **1A.7. Responsive/mobile**

**Duoi 1180px:**
- `.home-dashboard` va `.top-panels` chuyen ve 1 cot.
- Cot trai bo border phai.

**Duoi 1100px:**
- Header trang chu co the wrap, `min-height: 82px`.
- Menu chuyen xuong hang duoi, scroll ngang.

**Duoi 680px:**
- Header chung: `min-height: 56px`, `padding: 0 12px`.
- Header trang chu: `min-height: 86px`, `padding: 6px 12px 4px`.
- Brand text: `8px`; brand mark: `24px`.
- Header actions: `gap: 6px`.
- Icon button: `34px x 34px`, font `23px`.
- Card/grid chuyen ve 1 cot khi can.

### **1A.8. Quy tac khi chinh sua giao dien**

- Khong doi font, size chu, padding, margin, gap hoac border radius neu yeu cau chi la sua chuc nang.
- Neu can them CSS moi, uu tien selector hep theo khu vuc, vi du `.header-search`, `#prayers .prayer-card`, `.top-panels`.
- Tranh override rong len `.card`, `body`, `.section-block`, `.site-header` neu khong bat buoc.
- Sau khi doi CSS/JS, cap nhat version query string trong HTML de trinh duyet tai lai file moi.
- Sau moi lan chinh sua/toi uu, cap nhat lai file ghi chu nay.

### **3.5. Duyet loi cau nguyen**

**Chuc nang:**
- Hien thi cac loi cau nguyen nguoi xem gui len.
- Moi loi cau nguyen co:
  - Tieu de
  - Ho ten / Anonymous
  - Noi dung
  - Thoi gian gui
  - Trang thai cho duyet / da duyet
- Co thanh cuon de xem danh sach dai.
- Co nut Sua.
- Co nut Duyet hien thi.
- Co nut Xoa.

**Quy tac khi bam Sua:**
- Tieu de loi cau nguyen dien vao o "Mo ta ngan".
- Ho ten dien vao o "Thong tin phu".
- Ngay gio dien vao o "Ngay su kien".
- Noi dung loi cau nguyen dien vao o "Noi dung HTML day du".
- Giao dien sua dung chung form voi quan ly noi dung.

**Quy tac khi bam Duyet:**
- Luu loi cau nguyen thanh mot item trong danh muc Cau Nguyen.
- Trang thai bai noi dung la actived.
- Bai se hien tren website o muc Cau Nguyen.
- Luu lien ket giua loi cau nguyen goc va bai noi dung da tao.

**Quy tac khi bam Xoa trong man hinh Duyet loi cau nguyen:**
- Neu loi cau nguyen con cho duyet: xoa khoi danh sach duyet.
- Neu loi cau nguyen da duyet hoac da co bai noi dung lien ket: chi an khoi man hinh duyet, khong xoa bai trong danh muc Cau Nguyen.

## **11. LUONG NGHIEP VU TIEU BIEU**

### **11.1. Them bai viet moi**

1. Admin dang nhap trang quan ly.
2. Chon tab Noi dung.
3. Chon danh muc.
4. Nhap tieu de, mo ta ngan, noi dung HTML, thong tin phu, ngay, ngay tao, link anh, nguon.

## **15. DINH DANG JSON CHO KHAM PHA DUC TIN**

### **15.1. Cau truc file JSON bo cau hoi**

File JSON upload vao tab **Kham Pha Duc Tin** phai la mot mang JSON. Moi phan tu trong mang la mot cau hoi.

**Mau chuan:**

```json
[
  {
    "topic": "Truyen Tin",
    "question": "Trong bien co Truyen Tin, thien than nao da bao tin cho Duc Maria?",
    "options": [
      "Thien than Micae",
      "Thien than Raphael",
      "Thien than Gabriel",
      "Thien than Uriel"
    ],
    "answer": 2,
    "explanation": "Thien than Gabriel duoc Thien Chua sai den bao tin cho Duc Maria."
  }
]
```

### **15.2. Y nghia tung truong**

- **topic:** Ten chu de hoac moc su kien cua cau hoi. Vi du: Truyen Tin, Tham vieng ba Isave, Duc Me Hon Xac Len Troi.
- **question:** Noi dung cau hoi hien cho nguoi xem.
- **options:** Danh sach cac dap an trac nghiem. Nen co 4 dap an de giao dien dep va de hieu.
- **answer:** Vi tri dap an dung trong mang options, bat dau tu so 0.
- **explanation:** Noi dung giai thich hien ra sau khi nguoi xem tra loi dung.

### **15.3. Quy uoc dap an**

- **A = 0**
- **B = 1**
- **C = 2**
- **D = 3**

Vi du neu dap an dung la **C**, truong answer phai la:

```json
"answer": 2
```

### **15.4. Luu y khi chuyen tu Google Doc sang JSON**

- Moi cau hoi trong Google Doc nen co: tieu de cau, noi dung cau hoi, cac dap an A/B/C/D, dap an dung va giai thich.
- Khong dat dau phay sau phan tu cuoi cung trong mang JSON.
- Noi dung co dau tieng Viet duoc chap nhan.
- Nut **Dinh dang JSON** trong trang quan ly dung de kiem tra cau truc va can chinh lai JSON cho de doc.
- Sau khi bam **Dinh dang JSON**, van can bam **Luu bo cau hoi** de ghi du lieu vao Firestore.

14.108. Định dạng JSON cho game Hành trình Kinh Thánh
- Ngày ghi chú: 21/07/2026.
- Dữ liệu game được lưu trong Firestore tại collection siteSettings, document journeyBible.
- Cấu trúc gốc của document journeyBible:
```json
{
  "activeTopicId": "hanh-trinh-theo-dau-chan-chua-giesu",
  "topics": [
    {
      "id": "hanh-trinh-theo-dau-chan-chua-giesu",
      "title": "Hành trình theo dấu chân Chúa Giêsu",
      "label": "Tân Ước",
      "description": "Khám phá những cột mốc chính trong cuộc đời và sứ vụ của Chúa Giêsu.",
      "enabled": true,
      "pickerImageUrl": "https://...",
      "milestones": [],
      "challenges": {}
    }
  ]
}
```

Định dạng JSON cho màn hình chọn chủ đề:
- id: mã chủ đề, nên viết không dấu, dùng dấu gạch ngang. Ví dụ: hanh-trinh-theo-dau-chan-chua-giesu.
- title: tên chủ đề hiển thị trên card chọn chủ đề.
- label: nhãn nhỏ, ví dụ Tân Ước, Cựu Ước, Tin Mừng.
- description: mô tả ngắn cho chủ đề.
- enabled: true nếu cho hiển thị và chơi; false nếu muốn ẩn hoặc đang chuẩn bị.
- pickerImageUrl: URL ảnh card chọn chủ đề, lấy từ Cloudinary secure_url.

Định dạng milestones JSON:
```json
[
  {
    "number": 3,
    "title": "Chúa chịu phép rửa",
    "reference": "Mt 3,13-17",
    "region": "Sông Giođan",
    "scene": "baptism",
    "story": "Tại sông Giođan, Chúa Giêsu nhận phép rửa và bắt đầu bước vào sứ vụ công khai.",
    "lesson": "Người môn đệ được mời gọi sống khiêm nhường và lắng nghe tiếng Chúa Cha.",
    "cardImageUrl": "https://...",
    "x": 79.6,
    "y": 23.2
  }
]
```
- number: số cột mốc trên bản đồ, dùng để liên kết với thử thách.
- title: tên cột mốc hiển thị trong panel.
- reference: trích dẫn Kinh Thánh.
- region: địa danh hoặc khu vực.
- scene: mã cảnh dùng fallback CSS nếu chưa có ảnh card; ví dụ baptism, nativity, temple, desert.
- story: mô tả chính trong panel cột mốc.
- lesson: bài học/suy niệm ngắn trong panel.
- cardImageUrl: ảnh card nhỏ trên bản đồ, dùng Cloudinary secure_url.
- x, y: tọa độ phần trăm trên bản đồ. Có thể bỏ trống nếu dùng tọa độ mặc định trong code; chỉ nhập khi cần tinh chỉnh vị trí.

Định dạng challenges JSON:
```json
{
  "3": {
    "title": "Ghép Dấu Chỉ Bên Sông Giođan",
    "instruction": "Chọn đúng 3 dấu chỉ xuất hiện trong biến cố Chúa chịu phép rửa.",
    "verse": "Đây là Con yêu dấu của Ta, Ta hài lòng về Người.",
    "verseRef": "Mt 3,17",
    "rewardPoints": 50,
    "sceneImageUrl": "https://...",
    "targets": [],
    "options": []
  }
}
```
- Key "3" là số cột mốc, phải trùng với milestone.number.
- title: tên thử thách trong màn chơi.
- instruction: hướng dẫn người chơi.
- verse: câu Kinh Thánh mở khóa sau khi hoàn thành.
- verseRef: trích dẫn câu Kinh Thánh.
- rewardPoints: điểm đức tin cộng khi hoàn thành, chỉ cộng một lần.
- sceneImageUrl: ảnh nền/màn chơi riêng, dùng Cloudinary secure_url. Nếu bỏ trống, game dùng cảnh minh họa CSS fallback.
- targets: danh sách vị trí/dấu chỉ cần ghép đúng.
- options: danh sách lựa chọn người chơi bấm chọn.

Định dạng targets trong challenge:
```json
[
  {
    "signId": "water",
    "label": "Nước",
    "hint": "Dấu chỉ của phép rửa",
    "reveal": "Nước sông Giođan"
  },
  {
    "signId": "dove",
    "label": "Thánh Thần",
    "hint": "Ngự xuống trong hình chim bồ câu",
    "reveal": "Chim bồ câu"
  }
]
```
- signId: mã liên kết với option.id; phải trùng chính xác.
- label: tiêu đề ô cần ghép.
- hint: gợi ý khi chưa chọn đúng.
- reveal: nội dung hiện ra sau khi chọn đúng.

Định dạng options trong challenge:
```json
[
  {
    "id": "water",
    "label": "Nước sông",
    "text": "Dòng sông Giođan",
    "icon": "≈",
    "correct": true
  },
  {
    "id": "temple",
    "label": "Đền thờ",
    "text": "Không thuộc chặng này",
    "icon": "▥",
    "correct": false
  }
]
```
- id: mã lựa chọn. Nếu correct là true thì id phải trùng một target.signId.
- label: chữ lớn trên nút lựa chọn.
- text: mô tả nhỏ dưới label.
- icon: ký hiệu ngắn hiển thị trên lựa chọn.
- correct: true nếu là đáp án đúng, false nếu là đáp án nhiễu.

Quy trình nhập trong trang quản lý:
- Vào tab Hành trình Kinh Thánh.
- Chỉnh thông tin chủ đề, ảnh chọn chủ đề.
- Chọn cột mốc, nhập ảnh card map và text panel, bấm Cập nhật cột mốc vào JSON.
- Nhập ảnh/text màn chơi, targets/options, bấm Cập nhật màn chơi vào JSON.
- Bấm Lưu Hành trình Kinh Thánh để ghi vào Firestore.
- Nếu chỉ upload ảnh mà chưa bấm Cập nhật vào JSON và Lưu Hành trình Kinh Thánh thì dữ liệu chưa được lưu thật.

Lưu ý kỹ thuật:
- JSON phải dùng dấu ngoặc kép double quote, không dùng single quote.
- Không được có dấu phẩy dư ở phần tử cuối của array/object.
- Các URL ảnh nên dùng Cloudinary secure_url.
- Không thêm dữ liệu mẫu tự động ghi Firestore; code chỉ dùng fallback local khi Firestore chưa có journeyBible.
- Khi đổi cấu trúc JSON, phải kiểm tra cả admin.js, data.js và journey-test.js để đảm bảo backup, admin form và màn game cùng hiểu một schema.

### 14.162. Rà soát và tăng chi tiết Vercel Analytics toàn website
- Kiểm tra toàn bộ file HTML: các trang thật của website đều đã có script Vercel Insights `/ _vercel / insights / script.js`; riêng `googlee5213bf195fccdf7.html` là file xác minh Google nên giữ nguyên, không nhúng script để tránh ảnh hưởng xác minh.
- Nâng cấp `trackPageView()` trong `data.js`: mỗi lượt ghi analytics nội bộ Firebase cũng gửi thêm custom event `Kito Page View` sang Vercel, gồm key, label, kind, content_type, content_id và path.
- Thêm hàng đợi `window.vaq` trong helper `trackVercelEvent()` để custom event không bị mất nếu page JS chạy trước khi Vercel Insights tải xong.
- Bổ sung page view chi tiết cho trang `Khám Phá Đức Tin`, `Trang quản lý` và `Quản lý tài khoản`.
- Bổ sung custom event `Feedback Submitted` khi gửi góp ý thành công và `Prayer Request Submitted` khi gửi lời cầu nguyện thành công.
- Cập nhật cache `data.js` sang `20260723-vercel-analytics-events`; cập nhật cache các JS liên quan: `faith-discovery.js`, `gop-y.js`, `prayer-request.js`, `admin.js`, `accounts.js`.
